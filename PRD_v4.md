> **项目代号**：风生水起（FSSQ）
> **版本**：v4.0（产品化部署）
> **日期**：2026-05-14
> **定位**：精度优先的玄学+量化辅助决策工具，非娱乐产品
> **核心逻辑**：个人命盘（八字40%+紫微30%+奇门20%+占星10%）× 大势环境 → A股板块推荐 + 个股时效标注 + SnipeScore 8维量化评分

### 3.3 SnipeScore 8维评分体系

```
最终得分 = SnipeScore × 0.70 + 玄学匹配分 × 0.30 × 1.5

SnipeScore 8维：
├── 动量趋势 (15%)  — 5日/20日均线关系
├── 资金流向 (15%)  — 主力净流入/流出
├── 滞涨指标 (15%)  — 涨幅vs大盘偏离
├── 催化剂   (15%)  — 近期利好/利空
├── RSI      (10%)  — 14日RSI
├── 量比     (10%)  — 成交量比率
├── 趋势     (10%)  — SMA20趋势方向
└── 估值     (10%)  — PE/PB百分位
```

### 3.4 融合权重

```
融合五行向量 = {
    八字五行向量 × 0.40 +
    紫微五行向量 × 0.30 +
    奇门五行向量 × 0.20 +
    占星五行向量 × 0.10
} → 忌神MAX惩罚 → 奇门季节修正 → 板块映射 → 余弦相似度排序
```

### 4.2 核心结论格式（一句话总结）

```
命主{性别}，生于{出生年}年，日主{日干}{五行}（{强弱}），
喜用神为{用神}与{喜神}，当前大运{大运干支}（{年龄区间}）。
{目标年}年投资主线锁定{喜用神板块}（{板块1}、{板块2}为核心标的），
最佳窗口为{喜神}旺季节（如秋季9-11月），
最差时段为{忌神}旺月（如夏季6-7月），
应严守纪律、知行合一。
```

**案例**（1974男命，2026年）：
> 命主男，生于1974年，日主丁火（身旺），喜用神为土与金，当前大运乙丑（50-59岁）。
> 2026年投资主线锁定土金板块（银行、建筑建材为核心标的），
> 最佳窗口为金旺季节（如秋季9-11月），
> 最差时段为火水交战月（如夏季6-7月），应严守纪律、知行合一。

### 4.3 持仓策略卡片格式

每个推荐个股展示一张卡片，包含：
- 排位 + 股票名称 + 代码 + 板块 + 五行属性
- **操作时效日历**（12个月，每月一个圆点：🟢买入 / 🔵持有 / 🟠卖出 / ⚫空仓）
- **持仓策略建议**（3张卡片）：
  - 积极买入月份：列明具体月份
  - 持有观望月份：列明具体月份
  - 卖出/空仓月份：列明具体月份 + 持有/回避建议

### 4.4 西方占星小结格式

- 出生盘：太阳{星座}、月亮{星座}、上升{星座}
- 流年行星影响
- **水星逆行期**：具体日期范围（3段）
- **水逆实操建议**：不宜开新仓 / 不签合同 / 不换股 + 已持仓处理建议

### 4.5 报告尾部规范

- 风险提示（3条）
- [结束] 标记（居中，金色，加粗）
- 页脚：版本号 + 4-Agent架构 + 生成日期

## 五、CLI使用方式（当前可用）

### 4.1 orchestrator.py

```bash
python3 src/orchestrator.py \
  --birth "1974-07-05" \
  --hour 17 \
  --gender 男 \
  --place "开封" \
  --year 2026
```

| 参数 | 必填 | 说明 |
|------|------|------|
| `--birth` | ✅ | 出生日期 YYYY-MM-DD |
| `--hour` | ✅ | 出生时辰 0-23 |
| `--gender` | ✅ | 男/女 |
| `--place` | 可选 | 出生地（自动查经纬度） |
| `--year` | 可选 | 目标年份，默认当前年 |
| `--lat` | 可选 | 纬度（覆盖place推断） |
| `--lon` | 可选 | 经度（覆盖place推断） |
| `--current` | 可选 | 现居地 |
| `--output` | 可选 | 输出路径 |

### 4.2 run_pipeline.py

```bash
python3 run_pipeline.py \
  --birth 1974-07-05 \
  --hour 17 \
  --gender 男 \
  --place 开封 \
  --year 2026 \
  --current 旧金山 \
  --snipe 1
```

额外参数：`--snipe 0/1` 控制SnipeScore开关。

## 七、质量保障

### 6.1 自校验清单（代码内置ERROR_GUARDS）

```
每次生成报告前自动执行：

□ E001 大运年份覆盖：current_dayun.year_start ≤ target_year ≤ year_end
□ E002 大运方向：日干派（默认）→阳干男/阴干女→顺排，阴干男/阳干女→逆排；年干派→替换日干为年干
□ E003 起运年龄：二次独立计算交叉验证
□ E004 日柱万年历查表：禁止推算
□ E005 时柱五鼠遁查表：禁止手数步数
□ E006 农历检测：标注未转换则报错
□ E007 时辰缺失：默认子时+精度声明
□ E008 出生地：--place必填，经纬度自动查表
□ E009 月柱节气：内嵌节气精确时刻
□ E010 四级时效完整性：buy/hold/sell/empty齐全
```

### 6.2 SnipeScore数据源降级链

```
东方财富(push2) → 腾讯行情(qt.gtimg) → 新浪财经(hq.sinajs) → AKShare
│                  │                      │                     │
└──失败──────────→└──失败───────────────→└──失败──────────────→┘
                                                                  │
                                                     全部失败 → 报错
                                                     不允许降级玄学模式
```

### 6.3 Orchestrator容错

```
每个Agent独立try/except：
  astro-calc 失败 → 使用默认命盘结构（_get_default_astro）
  cosmic-trend 失败 → 使用默认宏观结构（_get_default_cosmic）
  fusion-engine 失败 → 使用默认融合结构（_get_default_fusion）
  star-hunter 失败 → 个股列表为空

最终状态：
  无错误 → SUCCESS
  有错误 → PARTIAL（列出哪些Agent降级）
```

## 十、安全与隐私

### 9.1 数据安全

- 用户出生信息（生辰八字、性别、出生地）属于**敏感个人信息**
- 当前版本**不存储任何用户数据**，纯计算型
- 未来Web版本需考虑：数据加密存储、访问鉴权、日志脱敏

### 9.2 报告声明

HTML报告应包含以下声明（v3.5已部分实现）：
1. **免责声明**：本报告基于传统术数理论，仅供参考，不构成投资建议
2. **精度声明**：时辰缺失时标注"按子时默认计算，存在精度偏差"
3. **数据来源声明**：SnipeScore数据来自东方财富/腾讯/新浪/AKShare
